## üìù Documentation Standards

When adding new documentation:
1. Choose the appropriate category directory
2. Use descriptive, capitalized filenames with underscores (e.g., `NEW_FEATURE_GUIDE.md`)
3. Include a clear title and table of contents for longer documents
4. Update this README with a link to your new document
5. Keep documentation up-to-date with code changes
